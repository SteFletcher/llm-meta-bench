How the three pillars are scored

Every number on the scoreboard traces to a public benchmark run by someone else — we aggregate, we don't re-test. This page documents exactly which benchmarks feed Planning, Security, and Coding, how each one produces its score, and the rules that keep the comparison honest.

00

The four rules

rule 1Every score is linked

Each cell on the scoreboard hyperlinks to the leaderboard, vendor announcement, or study it was taken from. If we can't cite it, we don't show it.

rule 2No blended composite

Models are tested on different benchmark subsets. Averaging Fable's SecPass against Opus's Cybench would compare different exams — so we rank within each benchmark and never publish a single "overall" number.

rule 3Proxies are daggered

Where only a predecessor model has been tested (Opus 4.6 on Vending-Bench, GPT-5.2 on Cybench), the score is shown with a † and the actual tested model named on hover. Proxy scores are evidence about a lineage, not the current model.

rule 4Agentic scores are system scores

The same Claude Opus 4 scores 64.9% on GAIA in the HAL harness and 57.6% in Open Deep Research — a 7-point swing from scaffolding alone. Every agentic number is a model+harness pair.

01

Planning — long-horizon agency

Planning means holding a goal over many steps: choosing tools, sequencing actions, recovering from surprises, and staying coherent over hours or a simulated year. Four benchmarks feed the pillar — and the honest headline is that this is the least-measured of the three.

planning · flagship

Vending-Bench 2 (Andon Labs)

andonlabs.com ↗

The model runs a simulated vending-machine business for one full simulated year: ordering stock, setting prices, negotiating with suppliers, and handling daily events.

Score
Final bank balance in dollars, starting from a $500 float. Higher = sustained coherent decision-making; models that lose the plot go bankrupt.
Current data
Claude Opus 4.6 leads at $8,017.59; Gemini 3 Pro at $5,478. Only 4 models have ever been evaluated.
Caveat: no current-generation flagship (Fable 5, Opus 4.8, GPT-5.6, Gemini 3.1) has been run. Both scoreboard entries are daggered predecessor proxies.
planning

Tau2-bench (Sierra)

pricepertoken.com ↗

Simulated customer-service agents in retail, airline, and telecom domains: the agent must use API tools and databases to resolve multi-turn requests while obeying company policy.

Score
Pass rate — a task counts only when fully resolved end-to-end. 226 models ranked.
Current data
GLM-5.2 leads at 99.1% (as of 2026-07-08).
Caveat: effectively saturated — a leader at 99.1% means the benchmark can no longer separate frontier models. Useful as a floor check, not a discriminator.
planning

GAIA (Princeton HAL)

hal.cs.princeton.edu ↗

General-assistant tasks requiring web research, multi-step reasoning, and tool use, run inside Princeton's HAL standardized agent harness.

Score
Task accuracy (%). We cite the scaffolded HAL board — the bare-model board strips agent scaffolding and reads very differently (GPT-5 Mini 44.8%).
Current data
Claude Sonnet 4.5 leads at 74.6%; Anthropic models hold the top six slots.
Caveat: the harness effect is the defining feature — the same model swings ~7 points between scaffolds. Read GAIA rankings as harness+model systems.
planning

ARC-AGI-2

arcprize.org ↗

Novel abstract-reasoning puzzles designed so memorization can't help — the closest thing to measuring the raw "figure out a plan for something you've never seen" skill.

Score
Percent of held-out tasks solved.
Current data
Gemini 3.1 Pro leads at 77.1%; GPT-5.4 reached 73.3% (83.3% for Pro).
02

Security — offense, defense, and safe output

Security splits into two very different questions: can the model do security work (CTFs, vulnerability analysis) and is the code it writes secure. The pillar uses one benchmark for each — and they disagree loudly.

security · flagship

Cybench, defensive subset (CoTool)

cotool.ai ↗

18 defensive tasks drawn from Cybench's 40 professional CTF challenges (HackTheBox, SekaiCTF, Glacier, HKCert). Defensive framing avoids the safety-classifier confound that offensive prompts trigger on safeguarded models.

Score
Percent of challenges solved unassisted.
Current data
Three-way tie at the 61% ceiling: Claude Opus 4.5, GPT-5.2, and GPT-5.2 Codex. GPT-5.2 gets there at $2.62/task — under a third of Opus 4.5's cost. Gemini 3.0 Pro 44%, Sonnet 4.5 33%.
Caveat: every entry is a predecessor generation — all scoreboard cells are daggered. Fable/Mythos-class models haven't been publicly run; Mythos 5 (safeguards lifted) is reputedly the strongest cyber model in existence but is restricted to Project Glasswing.
security

Endor Labs Agent Security League — SecPass

endorlabs.com ↗

200 real-world vulnerability-fix tasks from 108 open-source projects across 77 CWE classes, with cheating detection. FuncPass = the fix passes functional tests; SecPass = it also passes security tests. We chart SecPass — the number that measures whether AI-written fixes are actually safe.

Score
Percent of the 200 tasks where the fix passes security tests.
Current data
Claude Fable 5: 19.0% SecPass vs 59.8% FuncPass. Before Fable, the league-wide SecPass peak was 17.3% while FuncPass had climbed to 84.4%.
Caveat: over 80% of functionally-correct agent solutions contained security vulnerabilities. This is the pillar's most important number: the gap between "works" and "safe" has barely narrowed while raw coding skill soared.

Why not CyberSecEval? Meta's CyberSecEval 4 board currently lists a single evaluated model, and earlier versions are near-saturated — neither separates this model set, so it's excluded until coverage improves.

03

Coding — repo-scale software engineering

Coding is the best-instrumented pillar: current-generation models, live leaderboards, disclosed harnesses. Four benchmarks feed it, each covering a different slice — issue resolution, hard agentic work, terminal operation, and contamination-resistant problem solving.

coding · flagship

Terminal-Bench 2.1

codingfleet.com ↗

Agentic tasks executed in a real terminal: environment setup, debugging, builds, and multi-step operations. Scores are model+harness pairs (e.g. GPT-5.5 is scored inside Codex CLI).

Score
Percent of tasks completed.
Current data
GPT-5.6 Sol 88.8% (91.9% Ultra), Terra 87.4%, GPT-5.5 Codex 83.4%, Fable 5 83.1%, Sonnet 5 80.4%.
Caveat: Fable 5 was suspended from this board on 2026-06-12 and later reinstated — leaderboard governance is an active story, not a settled process.
coding

SWE-bench Pro

morphllm.com ↗

The harder successor to SWE-bench Verified: real repository issues selected to resist saturation and memorization. Currently the best single discriminator of frontier coding ability.

Score
Percent of issues resolved (patch passes the repo's tests).
Current data
Fable 5 80.3% (Anthropic-reported), Opus 4.8 69.2% — an 11-point gap where Verified shows only 6.
coding

SWE-bench Verified

benchlm.ai ↗

500 human-reviewed GitHub issues from popular Python repositories — the historical standard for AI software engineering.

Score
Percent of issues resolved.
Current data
Mythos 5 95.5%, Fable 5 95.0%, GPT-5.5 88.7%, Opus 4.8 88.6%.
Caveat: saturated and partially broken — an OpenAI audit found 59.4% of the hardest unsolved problems have flawed test cases. We keep it for continuity but treat it as directional only.
coding

LiveCodeBench

pricepertoken.com ↗

Rolling releases of fresh competitive-programming problems published after model training cutoffs — the contamination-resistant check on the other coding scores.

Score
Percent solved on the current rolling window.
Current data
Gemini 3 Pro Preview leads at 91.7% (as of 2026-07-12).

Refresh cadence: SWE-bench, LMArena, and LiveBench families are fetched by scripts/refresh_data.py; Terminal-Bench, Cybench, Vending-Bench, GAIA, and the Endor study have no machine-readable feeds and are updated by hand with citations.
