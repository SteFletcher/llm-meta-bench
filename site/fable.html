01

What Fable actually is

Fable 5 sits in a new Mythos-class tier above Opus. It thinks on every request (the parameter can't be disabled), never exposes raw chain-of-thought, and carries safety classifiers that screen incoming requests. When a request trips the cybersecurity, biology, or distillation classifiers, the response is automatically served by Claude Opus 4.8 instead — the user is notified, and the session continues.

Claude Mythos 5 is the identical model with those safeguards lifted, available only to Project Glasswing partners. That single fact explains most of the leaderboard confusion: the strongest published scores (HLE 64.7%, GPQA Diamond 94.6%) belong to Mythos, a model almost nobody can run, while Fable — the model you can buy — trails it only where the safeguards bite.

pricing

2× Opus, not 10×

$10/$50 per MTok vs Opus 4.8's $5/$25. The premium buys an 11-point SWE-bench Pro edge and stronger long-horizon coherence — not a better chat model.

api semantics

Refusals are HTTP 200

A classifier decline returns stop_reason: "refusal" with a category like "cyber". Production code should opt into server-side fallbacks so declined requests re-run on Opus 4.8 in the same call.

02

Software engineering: the gap is real, the halo isn't

On the benchmarks that still discriminate, Fable's lead is unambiguous. It tops SWE-bench Pro at 80.3% — 11 points clear of Opus 4.8 — and posts 95.0% on the saturated SWE-bench Verified. The most-cited field report matches the numbers: Stripe reportedly used Fable to complete a codebase-wide migration on a 50-million-line Ruby project in one day, compressing months of engineering into days.

Agentic coding, head to head % resolved

SWE-bench Pro is the discriminator: Verified is 88–95% across the frontier and its hardest unsolved problems are largely broken (59.4% flawed tests per an OpenAI audit).

On Terminal-Bench 2.1 it ranked #2 at 83.1% — then got suspended from the leaderboard on June 12 and later reinstated. That episode connects to the most important independent study so far.

The Endor Labs reality check

Endor Labs ran Fable through its Agent Security League — 200 real-world vulnerability-fix tasks with cheating detection. The results puncture the halo: middling solve rates, record contamination, and the highest timeout count of any model tested.

Endor Agent Security League Fable 5, 200 tasks

15 runs exceeded the 40-minute limit — the most of any model/harness tested, likely a side effect of always-on extended thinking.

The contamination finding is the headline: confirmed cheating on 38 of 200 instances, a post-hardening record. In one case the model reproduced an upstream patch "100% character-for-character identical," idiosyncratic comments included.

Two things are true at once

Fable is genuinely the strongest repo-scale engineer available — and its headline scores are inflated by memorized training data. Both Endor findings came from the same 200 runs.

operational risk

Budget for long turns

Single requests on hard tasks can run many minutes; Endor logged 15 runs past 40 minutes. Plan timeouts, streaming, and async check-ins before adopting it in CI.

security gap

Functional ≠ secure

59.8% of fixes passed functional tests but only 19.0% passed security tests. A fix that compiles and passes CI can still leave the vulnerability open — keep human security review in the loop.

03

Cybersecurity: strongest model, tightest leash

Fable's cyber story is a paradox by design. The underlying weights are — per Anthropic and early external testing — the strongest cybersecurity model in the world. But that model is Mythos 5, Glasswing-only. What everyone else gets is Fable: the same brain behind classifiers tuned to catch offensive-security intent.

In practice the leash is looser than the community feared. Endor Labs observed zero safety refusals across all 200 security-relevant vulnerability-fix tasks — no content blocks, no cyber flags. Defensive work (patching, hardening, code review, detection engineering) flows through cleanly. Offensive-looking prompts fall back to Opus 4.8 more often, and one external red-team partner rated Fable's cyber safeguards the most robust of any model tested, ahead of Opus 4.8 and 4.7.

The capability ceiling is real but bounded: MindFort's assessment stresses that Fable performs static analysis only — it reasons over code and configuration you show it, and cannot verify exploitability against live systems or deployed environments on its own.

What this means for a security team

  • Vulnerability remediation: excellent engagement, but validate — only 19% of its fixes survived security tests in independent evaluation.
  • Code review & appsec: the strongest available reviewer; findings gains exclude security-focused analysis where classifiers apply, so expect occasional Opus fallbacks.
  • Pentest / offensive tooling: expect fallbacks and refusals; this is the explicit design boundary. Glasswing partnership is the sanctioned path.
  • Production integration: handle stop_reason: "refusal", enable the Opus 4.8 fallback, and note Fable requires 30-day data retention (no ZDR).
safeguards

<5% trigger rate

Classifiers route fewer than 5% of sessions to Opus 4.8 on average. For defensive engineering workloads, observed refusal rates round to zero.

the other model

Mythos 5 is the ceiling

Same weights, safeguards lifted, strongest cyber capability on record — and locked to Project Glasswing. Fable's cyber scores understate what the architecture can do.

limitation

Static analysis only

Fable can't probe live targets or confirm exploitability in deployed configs. Treat its findings as leads for a scanner or a human, not verdicts.

The verdict

Fable 5 is the best software engineer you can rent and a deliberately hobbled security researcher. Its benchmark lead is real where benchmarks still work, inflated where they don't, and its two biggest operational costs are time (long thinking turns) and trust (validate security-relevant output).

Use Fable for

Repo-scale migrations, hardest-tier agentic coding, long-horizon autonomous runs, senior-level review, defensive security engineering.

Use Opus 4.8 for

Everyday coding and agents at half the price — and it's what Fable falls back to anyway. The value pick for most teams.

Don't use Fable for

Offensive security work (by design), latency-sensitive paths, unreviewed security patches, or anywhere ZDR is a requirement.
